The Requirements:

Build a search bar for suggested words. As the user types, call an API and display the results in a list.

  • Don't call the API on every single keystroke (wait 300ms).
  • If request A is sent, then request B is sent, but request A finishes after B, you must ignore A so the UI doesn't show "stale" data.
  • Display a "Loading..." state and a list of results.

The Imperative React Solution: [Link to Code]

  • Debounce: setTimeout is used to manually handle debouncing, performing a search after the user has stopped typing for at least 300 ms.
  • Cancellation: An AbortController cancels ongoing fetch requests and a local active flag ensures only the most recent request updates the state. We also need to manually clean up the timer with clearTimeout.
  • State: React uses useState for tracking search strings and results, and useMemo to derive the component's visual state. The solution is render-based because every stat change triggers a re-run of the entire function. More care is needed to reset suggestions to null when a search starts to prevent showing a stale or empty state between changing the state from loading and setting the results.
  • An initial state requiring at least 3 characters before searching as well as loading, loaded, and empty state.
  • A custom caching layer for instant data retrieval.

The Declarative Angular Solution: [Link to Code]

  • Debounce: debounceTime(300) limits API calls, performing a search after the user has stopped typing for at least 300 ms.
  • Cancellation: switchMap allows automatic API request cancellation if a new search term is entered, preventing stale data from being displayed.
  • State: Use of Angular Signals & Computed is push-based, which prevents showing a stale or empty state between changing the state from loading and setting the results.
  • An initial state requiring at least 3 characters before searching as well as loading, loaded, and empty state.
